2. HAVE YOU BUILT APPLICATIONS USING OPENAI OR AI IMAGE ANALYSIS?
--------------------------------------------------------------------------------

Yes — the demo site above was built specifically for this project and
demonstrates direct hands-on experience with the exact stack you need.

What went into building the demo:

  - GPT-4o Vision integration: the backend sends uploaded images to OpenAI's
    Vision API with a structured prompt, receiving JSON with labels, description,
    objects, and attributes. A second GPT-4o call then generates 5 ranked
    recommendations based on that analysis.

  - Async FastAPI backend: built with Python's async/await throughout so image
    uploads, AI calls, and database writes happen concurrently without blocking.

  - Supabase Storage pipeline: images upload to a cloud bucket, generating a
    public URL passed directly to the OpenAI Vision API.

  - Structured JSON outputs: AI is prompted to return strict JSON validated
    against Pydantic models, ensuring consistent and type-safe data.

  - 91 automated tests: full pipeline covered by unit and integration tests
    including mocked GPT-4o calls to verify model format and field presence.

The demo site is the example. Upload any image to see it in action:

  Rate Limiting      | Redis (Upstash)           | Serverless, free tier
  Frontend Hosting   | Vercel                    | Zero-config Next.js, free tier
