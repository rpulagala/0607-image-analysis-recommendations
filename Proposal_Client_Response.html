02

Have You Built Applications Using OpenAI or AI Image Analysis?

Yes — the demo site you just saw was built specifically for this project and demonstrates hands-on experience with exactly the stack you need.

What went into building the demo:

  • GPT-4o Vision integration — The backend sends uploaded images directly to OpenAI's Vision API with a structured prompt, receiving a JSON response with labels, description, detected objects, and attributes. A second GPT-4o call then uses that analysis to generate 5 ranked recommendations.
  • Async FastAPI backend — Built with Python's async/await throughout so image uploads, AI calls, and database writes happen concurrently without blocking.
  • Supabase Storage pipeline — Images are uploaded to a cloud storage bucket, generating a public URL that is passed directly to the OpenAI Vision API.
  • Structured JSON outputs — The AI is prompted to return strict JSON objects validated against Pydantic models, ensuring consistent and type-safe data across the pipeline.
  • 91 automated tests — The entire pipeline is covered by unit and integration tests including mocked GPT-4o calls to verify model format, field presence, and recommendation caps.
The demo site is the example. Sign up at https://0607-image-analysis-recommendations.vercel.app and upload any image to see the full AI pipeline end to end.
